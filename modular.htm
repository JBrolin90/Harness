Let’s build the Hazel-Controller (H-C). To keep this modular and "Unix-like," we will use Python to handle the Ollama API, and we’ll delegate all file actions (the "Hands") to separate Bash scripts. This way, your Python controller only ever sees the result of the action, never the complex JSON-shell-parser noise. The Modular Architecture controller.py: The "Brain." It tracks conversation history and sends raw prompts to Ollama. tools/ directory: A folder containing your scripts (read.sh, write.sh, boiler_fix.sh). memory/ directory: Stores the agent's context and system prompts. Phase 1: The Controller Skeleton We will start by creating a simple Python script that talks to Ollama without the JSON-parsing baggage. You'll need the requests library (pip install requests). Python import requests import json def call_ollama(prompt, context): url = "http://localhost:11434/api/generate" payload = { "model": "qwen2.5-coder:14b", "prompt": f"Context: {context}\n\nUser: {prompt}", "stream": False } response = requests.post(url, json=payload) return response.json()['response'] # Basic loop for your terminal context = "You are Hazel, Home Assistant expert." while True: user_input = input("Joachim: ") if user_input.lower() == "exit": break # 1. Send prompt response = call_ollama(user_input, context) print(f"Hazel: {response}") Why this is the "General Fix" No JSON-Constraint: The api/generate endpoint returns pure text. If you want a code block, the model gives you a code block. There is no middle-man "Tool Parser" killing your session if it doesn't like your YAML formatting. Deterministic Tools: Instead of asking the model to "output JSON to use a tool," you will eventually add a step in the loop: If the LLM's response contains a keyword like EXEC_BOILER_FIX, your Python script calls the bash script.
