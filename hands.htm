Now that we have a working "Brain" (the Ollama API call), we need to give Hazel her "Hands." In the pi shell, this was done by forcing the model to output a massive JSON object. We are going to do something much more resilient: Command Parsing. We will teach Hazel a very simple text-based protocol. If she needs a file, she will just type a command like !READ /path/to/file, and your Python script will intercept it, run the bash command, and feed the text back to her. Phase 2: The Action Loop We are going to update controller.py to include a tool execution step and an internal feedback loop. Replace your controller.py with this updated version: Python import requests import subprocess import re def call_ollama(prompt, context): url = "http://localhost:11434/api/generate" payload = { "model": "qwen2.5-coder:7b", # Swapped to 7B as requested "prompt": f"{context}\n\n{prompt}", "stream": False } response = requests.post(url, json=payload) return response.json()['response'] def execute_tool(command, arg): """The 'Hands': Executes local commands based on LLM requests.""" print(f"\n[🔧 Harness executing: {command} on {arg}]") if command == "!READ": try: # Using basic 'cat' to read files result = subprocess.run(["cat", arg.strip()], capture_output=True, text=True) if result.returncode == 0: return f"[SYSTEM OUTPUT: Content of {arg}]\n{result.stdout}" else: return f"[SYSTEM ERROR: Could not read {arg}]\n{result.stderr}" except Exception as e: return f"[SYSTEM ERROR: {str(e)}]" return "[SYSTEM ERROR: Unknown command]" # 1. The System Prompt establishes the rules of the new harness system_prompt = """You are Hazel, a Home Assistant expert. You have access to a local file system via your Harness. To read a file, you MUST output exactly this format on a new line: !READ /path/to/file Do not use JSON to call tools. Just use the !READ command. Wait for the system to provide the file content before giving your final answer. """ conversation_history = "" print("Hazel-Harness initialized. Type 'exit' to quit.") while True: user_input = input("\nJoachim: ") if user_input.lower() == "exit": break # Append user input to history conversation_history += f"\nJoachim: {user_input}\nHazel: " # 2. Call the LLM response = call_ollama(conversation_history, system_prompt) print(f"Hazel: {response}") # Append Hazel's initial response to history conversation_history += response # 3. Intercept Tools (The Magic Trick) # Check if Hazel tried to use the !READ command match = re.search(r'!(READ)\s+(.+)', response) if match: tool_cmd = f"!{match.group(1)}" tool_arg = match.group(2) # Execute the local bash command via Python system_result = execute_tool(tool_cmd, tool_arg) # 4. The Auto-Loop: Feed the file content directly back to Hazel print("\n[Harness feeding file content back to Hazel...]") conversation_history += f"\n{system_result}\nHazel: " # Call the LLM again with the new file context so she can actually answer your question final_response = call_ollama(conversation_history, system_prompt) print(f"Hazel: {final_response}") # Save final answer to history conversation_history += final_response Why this architecture changes everything: The ReAct Loop (Reason and Act): When Hazel needs a file, she asks for it. The script intercepts her request, reads the file using standard Linux subprocess.run(), and automatically prompts her again with the file's contents. You don't have to copy/paste anything. Invisible Enforcement: The LLM isn't fighting a parser. It just types !READ /etc/hostname and gets the text back. If she messes up the formatting slightly, it just prints as text—it doesn't crash your whole terminal. State Memory: conversation_history is now acting as your rolling context window, allowing her to remember what you asked two steps ago. The Test Run this script.
