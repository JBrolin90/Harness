### Hazel-Controller: Autonomous ReAct Harness The final implementation of the Python controller completely decouples the reasoning engine from the shell's execution environment. By using a text-based command protocol (`!READ`, `!WRITE`) instead of forced JSON schemas, the agent can output multi-line configurations (like YAML) without triggering buffer constraints or parsing errors. #### Key Mechanisms 1. **Bulletproof Parsing:** Uses `re.DOTALL` to capture multi-line code blocks wrapped in `~~~`, effectively bypassing UI rendering bugs and ignoring minor formatting inconsistencies. 2. **The Autonomous Inner Loop:** A nested `while` loop that intercepts tool requests, executes local bash commands, and silently feeds the output back into the context window. This creates a true ReAct (Reason and Act) loop, allowing the agent to chain multiple read/write actions until the task is fully resolved. #### The Working Code (`controller.py`) ```python import requests import subprocess import re import os def call_ollama(prompt, context): url = "http://localhost:11434/api/generate" payload = { "model": "qwen2.5-coder:7b", "prompt": f"{context}\n\n{prompt}", "stream": False } try: response = requests.post(url, json=payload) response.raise_for_status() return response.json()['response'] except Exception as e: return f"[API CONNECTION ERROR: {str(e)}]" def execute_tool(command, arg, content=""): """The 'Hands': Executes local commands based on LLM requests.""" print(f"\n[🔧 Harness executing: {command} on {arg}]") if command == "!READ": try: result = subprocess.run(["cat", arg.strip()], capture_output=True, text=True) if result.returncode == 0: return f"[SYSTEM OUTPUT: Content of {arg}]\n{result.stdout}" else: return f"[SYSTEM ERROR: Could not read {arg}]\n{result.stderr}" except Exception as e: return f"[SYSTEM ERROR: {str(e)}]" elif command == "!WRITE": try: safe_path = arg.strip() # Explicit file writes; verify absolute paths if using SSH/containerized terminals with open(safe_path, 'w') as f: f.write(content.strip()) return f"[SYSTEM OUTPUT: Successfully wrote {len(content)} characters to {safe_path}]" except Exception as e: return f"[SYSTEM ERROR: Could not write to {safe_path}: {str(e)}]" return "[SYSTEM ERROR: Unknown command]" # 1. System Prompt establishes the rules of the harness system_prompt = """You are Hazel, a Home Assistant expert. You have access to a local file system via your Harness. AVAILABLE TOOLS: 1. Read a file: !READ /path/to/file 2. Write a file: !WRITE /path/to/file ~~~yaml [YOUR FILE CONTENT HERE] ~~~ RULES: - Do not use JSON to call tools. Use the exact text commands above. - When using !WRITE, the file path must be on the same line, followed immediately by a block wrapped in ~~~ (tildes). - Wait for the system to confirm file operations before concluding. """ conversation_history = "" print("Hazel-Harness initialized. Type 'exit' to quit.") while True: user_input = input("\nJoachim: ") if user_input.lower() == "exit": break conversation_history += f"\nJoachim: {user_input}\nHazel: " # 2. Get initial response current_response = call_ollama(conversation_history, system_prompt) print(f"Hazel: {current_response}") conversation_history += current_response # 3. The Autonomous Tool Loop (ReAct) while True: system_result = None # Check for !READ command (stops at end of line) read_match = re.search(r'!(READ)\s+([^\n]+)', current_response) # Check for !WRITE command (bulletproof multi-line capture) write_match = re.search(r'!(WRITE)\s+([^\n]+)\n+~~~[^\n]*\n(.*?)~~~', current_response, re.DOTALL) if write_match: tool_cmd = f"!{write_match.group(1)}" tool_arg = write_match.group(2) tool_content = write_match.group(3) system_result = execute_tool(tool_cmd, tool_arg, tool_content) elif read_match: tool_cmd = f"!{read_match.group(1)}" tool_arg = read_match.group(2) system_result = execute_tool(tool_cmd, tool_arg) # 4. If a tool was used, feed it back and loop again if system_result: print(f"\n[Harness feeding system result back to Hazel...]") conversation_history += f"\n{system_result}\nHazel: " # Agent processes the tool output and generates next action current_response = call_ollama(conversation_history, system_prompt) print(f"Hazel: {current_response}") conversation_history += current_response else: # Task complete; return control to user break ```
